# 트레이더 필수 기능 확장 계획

**Goal:** 실제 한국 주식 트레이더들이 원할 만한 기능을 우선순위별로 정리. 현재 데이터 인프라로 구현 가능한 것부터 단계적 추가.

---

## 🎯 현재 있는 것 (리마인드)

- ✅ 20개 프리셋 기술적 검색기
- ✅ 종목 상세 + 5년 차트 + MA/RSI/MACD/BB/볼륨
- ✅ 구매 추천 일괄 뷰 (프리셋 매칭)
- ✅ 입문 추천 종목 (5개 섹션)
- ✅ 전체 종목 검색
- ✅ 밥그릇 패턴 (Beta) + 거래량 품질 점수

---

## 🏆 Tier 1: 필수 (핵심 트레이더 니즈)

모든 주식 앱이 기본 제공. 구현 시 체감 가장 큼.

### ① ⭐ 관심종목 (Watchlist)
**무엇**: 자주 보는 종목 북마크. 로그인 후 내 리스트 바로 확인.
**왜**: 2,500종목 중에서 매일 내가 관심 있는 5~20개만 보고 싶음.
**구현**:
- 종목 상세 페이지에 "⭐ 관심종목 추가" 버튼
- 새 페이지 `/watchlist` — 내 관심종목 목록 + 오늘 등락률
- **localStorage 저장** (2~3명 쓸 거라 DB 불필요)
- 추후 JWT 쿠키에 user ID 넣어 per-user 분리 가능
- 예상 구현: **3~4시간**

### ② 🔔 가격·지표 알림
**무엇**: "삼성전자가 70,000원 되면 알려줘" 또는 "MA20 돌파하면 알려줘".
**왜**: 매일 수동으로 찾지 않아도 됨. 매수·매도 타이밍 포착.
**구현**:
- 종목 상세에 "알림 설정" 버튼
- 알림 타입: 가격 위/아래 / MA20 돌파 / RSI 30 이하 / 골든크로스
- **브라우저 Notification API** (푸시 알림)
- 데이터 갱신 워크플로가 알림 조건 체크 후 브라우저로 전달
- 한계: 사이트가 열려있어야 알림 (Service Worker 쓰면 백그라운드도 가능하지만 복잡)
- 대안: **이메일 알림** (SendGrid 무료 월 100건)
- 예상 구현: **1~2일**

### ③ 📊 포트폴리오 추적
**무엇**: 보유 종목 + 매수가 + 수량 입력 → 현재 수익률 자동 계산.
**왜**: 내가 뭘 얼마에 샀는지 기억 안 남. 수익률 한눈에.
**구현**:
- `/portfolio` 페이지
- 각 행: 종목명, 매수가, 수량, 현재가, **수익률 %**, **손익**
- 총 포트폴리오 가치 + 전일 대비 변화
- **localStorage 저장**, JSON export/import
- 섹터별 분산 비율 차트 (도넛)
- 예상 구현: **4~5시간**

### ④ 📓 매매일지 (Trading Journal)
**무엇**: 매수/매도 기록 + 왜 샀는지 메모.
**왜**: 내 투자 습관 반성. 성공/실패 패턴 발견.
**구현**:
- `/journal` 페이지
- 입력: 날짜 / 종목 / 매수·매도 / 가격 / 수량 / 사유 / 프리셋 태그
- 필터: 종목별, 기간별, 결과(수익/손실)
- 통계: 월별 수익률, 승률, 평균 보유 기간
- 예상 구현: **1일**

## 🎨 Tier 2: 고가치 (실사용 시 자주 원하게 되는 기능)

### ⑤ 🔄 종목 비교 (차트 오버레이)
**무엇**: 삼성전자 vs SK하이닉스 수익률 그래프 겹쳐서 비교.
**왜**: "같은 기간 누가 더 잘 올랐는가" 비교하며 섹터 주도주 파악.
**구현**:
- 비교할 종목 2~4개 선택
- 시작일 기준 **정규화 수익률**(%) 곡선 오버레이
- 현재 ohlcv/{code}.json 데이터로 가능
- 예상: **5~6시간**

### ⑥ 🧪 간단 백테스팅
**무엇**: "이 프리셋을 6개월 전부터 매일 적용했다면 평균 수익률?"
**왜**: 프리셋 신뢰도 숫자로 확인. 사용 여부 판단.
**구현**:
- 프리셋 선택 + 기간 선택 (3개월/6개월/1년)
- 과거 각 날짜에 프리셋 매칭된 종목 → D+N 평균 수익률 집계
- 이미 `pattern_stats.json`에 유사 통계 있음. 확장
- 결과: 총 수익률 / 승률 / 최대 낙폭 / 평균 보유 기간
- 예상: **1~2일**

### ⑦ 🎨 섹터 히트맵
**무엇**: 전체 섹터(반도체/2차전지/바이오 등) 오늘 등락률을 색상 타일로.
**왜**: "오늘 주도 섹터는 뭔지" 한눈에.
**구현**:
- `/heatmap` 페이지
- 각 섹터 타일 (크기 = 시가총액 합, 색상 = 등락률)
- `sectors.json` + 종가 등락 기반
- 클릭 시 해당 섹터 종목 리스트
- Tree map 라이브러리 (d3) 필요
- 예상: **1일**

### ⑧ 📉 공매도 정보
**무엇**: 공매도 잔고 비율, 공매도 급증 종목 리스트.
**왜**: 하락 베팅 세력 감지. 단타 역접근.
**구현**:
- Python 파이프라인에 `pykrx.get_shorting_balance_top50` 등 추가
- 새 데이터 `shorting.json` 일일 수집
- `/shorting` 페이지 — 공매도 잔고 Top 50
- 예상: **1일**

### ⑨ 💰 투자자별 동향 차트
**무엇**: 종목 상세 페이지에 외국인·기관 일별 순매수 그래프.
**왜**: "누가 매집 중인지" 시각적으로.
**구현**:
- 종목 상세 페이지에 막대그래프 추가
- `foreign_net` + `institution_net` 10일치 이미 수집 중 → 30일로 확장
- 차트 컴포넌트 1개 추가
- 예상: **4시간**

### ⑩ 🔧 커스텀 스크리너
**무엇**: 사용자가 여러 프리셋 + 파라미터 조합을 **저장**하고 이름 붙이기.
**왜**: "나만의 검색식" — 매일 같은 조건 빠르게 실행.
**구현**:
- 검색기 페이지에 "현재 조건 저장" 버튼
- `/custom` 페이지 — 저장된 검색식 리스트
- AND/OR 조건 UI
- localStorage 저장
- 예상: **1일**

## 🚀 Tier 3: 멋짐 (있으면 좋지만 복잡)

### ⑪ 📰 뉴스·공시 통합
**무엇**: 종목별 최근 공시·뉴스 타임라인.
**왜**: 급등/급락의 이유 즉시 파악.
**구현**:
- **DART OpenAPI** (무료, 한국 전자공시) 연동 → 공시
- Naver 검색 API 또는 RSS → 뉴스
- 종목 상세에 "최근 이슈" 탭 추가
- 예상: **2~3일**, 외부 API 의존

### ⑫ 📅 경제·실적 캘린더
**무엇**: FOMC / 실적발표 / 배당락 / 공휴일 캘린더.
**왜**: 이벤트 전후 변동성 대비.
**구현**:
- 실적발표 — DART에서 공시 "연결재무제표"
- FOMC — 외부 API (fed-calendar 등)
- `/calendar` 페이지 or 헤더 뱃지
- 예상: **1~2일**

### ⑬ 📸 스크린샷 공유
**무엇**: 차트/추천 결과를 PNG로 저장 → 카톡 공유.
**왜**: 친구에게 종목 추천 보여주기.
**구현**:
- `html2canvas` 또는 `html-to-image` 라이브러리
- 차트 영역 캡처 버튼
- 다운로드 or 클립보드
- 예상: **3시간**

### ⑭ 🎯 차트에 프리셋 신호 마커
**무엇**: 차트에서 "2026-03-15 골든크로스 ▲" 화살표 표시.
**왜**: 과거 어느 시점에 어떤 신호가 있었는지 시각적 학습.
**구현**:
- `calculate_pattern_stats.py`가 이미 과거 이벤트 스캔
- 차트 overlay marker 렌더링 (lightweight-charts 지원)
- 예상: **5~6시간**

### ⑮ 💼 모의투자 게임
**무엇**: 가상 자금 1천만원으로 실전처럼 매매.
**왜**: 리얼 머니 전에 전략 연습.
**구현**:
- `/sandbox` 페이지
- 매수/매도 시뮬레이션
- 일일 평가 손익 계산
- localStorage 저장
- 예상: **2~3일**

**구현**: 종목 간 가격 상관계수 pre-compute, 예상 **1일**
