User Verification Checklist (Task 12):
  .env.local 작성 후 `npm run dev` 로 아래 시나리오 확인:
  
  1. `http://localhost:3000/` → 404
  2. `http://localhost:3000/wrongpath` → 404
  3. `http://localhost:3000/dev/login` → 로그인 폼
  4. `http://localhost:3000/dev/screener` (쿠키 없이) → `/dev/login` 리다이렉트
  5. 로그인 (체크박스 미체크) → "약관에 동의해주세요" 에러
  6. 로그인 (잘못된 비번) → "아이디 또는 비밀번호가 일치하지 않습니다"
  7. 로그인 (올바른 ID/PW, 체크 O) → `/dev/screener` 리다이렉트 (임시 404 — Phase 4까지 미구현이므로 정상)
  8. DevTools → Application → Cookies에 httpOnly `auth_token` 존재 확인
  9. 로그아웃 API 호출 시 쿠키 삭제
  10. 잘못된 비번 6회 연속 → 6번째부터 429 "시도 제한" 응답
  11. `.env.local`에서 `SITE_ENABLED=false` 변경 후 서버 재시작 → 모든 경로 503 유지보수 페이지
  12. 다양한 쿠키 (위조, 만료) 삽입 → 로그인 리다이렉트

  .env.local 예시:
  ```
  JWT_SECRET=dev-secret-at-least-32-bytes-long-for-HS256-ok
  SECRET_BASE_PATH=dev
  USER_KYUNGWON_ID=kyungwon
  USER_KYUNGWON_HASH=<npm run hash-password -- "test1234" 출력 해시>
  USER_OH_ID=oh
  USER_OH_HASH=<동일>
  SITE_ENABLED=true
  ```
